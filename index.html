# Tools * [GitHub](http://www.github.com) * [Slack](http://www.slack.com) * [ShareLatex](http://www.sharelatex.com)

## git vs GitHub * ```git``` is an open-source distributed version control system * developed for Linux kernel development * GitHub is a (commercial) service built around ```git```
## Important git Features * flexible branching model * distributed * supports staging
## Key git Commands * ```init``` and ```clone``` * ```status``` and ```diff``` * ```add``` and ```commit``` * ```push``` * ```pull``` * ```branch```
# git init ```init``` initializes a new repository. I usually do this on GitHub.
# git clone ```clone``` copies a repository into a new working directory. ``` > git clone git@github.com:ngehlenborg/collaboration-tutorial.git Cloning into 'collaboration-tutorial'... remote: Counting objects: 5, done. remote: Compressing objects: 100% (4/4), done. remote: Total 5 (delta 0), reused 0 (delta 0), pack-reused 0 Receiving objects: 100% (5/5), done. Checking connectivity... done. ```
## git status ```status``` shows the current status of the working directory with respect to the repository. ``` > git status On branch master Your branch is up-to-date with 'origin/master'. nothing to commit, working directory clean ```
## git status (cont'd) Then I added some files to the directory ... ``` > git status On branch master Your branch is up-to-date with 'origin/master'. nothing to commit, working directory clean octothorpe:collaboration-tutorial nils$ git status On branch master Your branch is up-to-date with 'origin/master'. Untracked files: (use "git add ..." to include in what will be committed) .DS_Store css/ index.html js/ lib/ plugin/ nothing added to commit but untracked files present (use "git add" to track) ```
## git add ```add``` stages changes before they are committed to the repository. ``` > git status On branch master Your branch is up-to-date with 'origin/master'. nothing to commit, working directory clean ```
